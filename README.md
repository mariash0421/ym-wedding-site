# 結婚式ウェブサイト

このプロジェクトは結婚式のための簡単なウェブサイトです。前撮り写真、動画、当日のアジェンダ、式当日の写真を共有できます。

## プロジェクト構成

```
wedding/
├─ index.html         # トップページ（前撮り、動画、当日アジェンダへの導線）
├─ gallery.html       # ギャラリーページ（photos.jsonから動的に読み込み）
├─ agenda.html        # 当日のアジェンダ
├─ photos.json        # 写真のマニフェストファイル
├─ css/
│  └─ styles.css      # スタイルシート
├─ js/
│  └─ app.js          # JavaScriptファイル（ギャラリー機能）
├─ media/
│  ├─ prephoto/       # 前撮り写真を格納
│  ├─ eventphoto/     # 当日写真を格納（式後に追加）
│  └─ video/          # 動画ファイルを格納
└─ assets/
   └─ logo.png        # ロゴ画像（必要に応じて追加）
```

## 使い方

### 1. 前撮り写真の追加

1. 写真を `media/prephoto/` ディレクトリに配置
2. `photos.json` の `"pre"` 配列に写真情報を追加

```json
{
  "pre": [
    {"url":"media/prephoto/01.jpg","thumb":"media/prephoto/01-thumb.jpg","title":"前撮り1"},
    {"url":"media/prephoto/02.jpg","thumb":"media/prephoto/02-thumb.jpg","title":"前撮り2"}
  ]
}
```

### 2. 動画の追加

YouTube動画の場合は `photos.json` の `"video"` 配列にURLを追加：

```json
{
  "video": [
    {"url":"https://www.youtube.com/watch?v=XXXX","title":"前撮り動画"}
  ]
}
```

### 3. 当日写真の追加（式後）

式後に撮影した写真を追加する手順：

1. 写真を `media/eventphoto/` ディレクトリに配置
2. `photos.json` の `"event"` 配列に写真情報を追加

```json
{
  "event": [
    {"url":"media/eventphoto/01.jpg","thumb":"media/eventphoto/01-thumb.jpg","title":"挙式"},
    {"url":"media/eventphoto/02.jpg","thumb":"media/eventphoto/02-thumb.jpg","title":"披露宴"}
  ]
}
```

### 4. アジェンダのカスタマイズ

`agenda.html` を編集して、実際のスケジュールに合わせて時間とイベント内容を変更してください。

### 5. ロゴの追加

`assets/logo.png` にロゴ画像を配置してください。画像がない場合は `index.html` から該当行を削除してください。

## 公開方法

### QRコードでの共有

1. このフォルダをウェブサーバーにアップロード（GitHub Pages、Netlify、Vercelなど）
2. 公開URLをQRコードに変換
3. 招待状や会場にQRコードを配置

### ローカルでのテスト

ローカル環境で確認する場合：

```bash
# Pythonがインストールされている場合
python3 -m http.server 8000

# Node.jsがインストールされている場合
npx http-server
```

ブラウザで `http://localhost:8000` にアクセスして確認できます。

トップページの確認では `index.html` の「Photo Gallery」セクションを見ると、2枚表示の自動スライドを確認できます。
初期状態では1枚目と2枚目が表示され、約3.6秒ごとに3枚の画像が同時に左へ移動し、次の2枚へ切り替わります。
このサイトはスマートフォン閲覧を前提にしているため、トップページのフォトギャラリーもモバイル幅で2枚が見やすく収まるよう調整しています。
スライドは3枚を同時に一定速度で動かし、左の画像が抜けるタイミングと次の画像が入るタイミングが揃うようにしています。
トップページのローディング画像は `object-fit: cover` で表示し、端末の画面比率に合わせて写真を自動的にトリミングしながら画面全体を覆います。
トップページのローディング画像は、最初のアクセス時とブラウザ更新時のみ表示し、同じタブ内で別ページからトップへ戻った場合は描画前から完全に非表示にする仕様です。
固定ヘッダーは左に `Yuki & Mashiro`、右にアイコンと楕円の `MENU` ボタンを配置し、`MENU` ボタンでナビゲーションを開閉する仕様です。
ヘッダー背景は、上端に細い白線を入れつつ、その下だけ半透明の白が重なり、下に向かって透明へ抜けるグラデーションにしています。
トップページの `Profile` から `Seat Map` までの案内文と目次ラベルは、英語表記で統一しています。
セクション1の目次には `Wedding Video & Ceremony Photos` も含めており、クリックすると該当セクションまでスムーズスクロールします。
`Greeting` は、青い帯の見出しセクションと、背景画像上に本文だけを載せるメッセージセクションに分けています。
セクション7の `Course Menu & Drinks` は、フードとドリンクの両方を参考メニュー表の雰囲気に寄せた白ベースの紙面デザインで構成しています。フードは縦に流れるコース表示、ドリンクは左右2列でカテゴリを分けた一覧表示にしており、スマートフォンでも見やすい形です。
セクション6の `Timetable` は画像表示にしており、スマートフォンでは画像をタップすると拡大表示できます。
セクション4の `Profile` は、雄己・真白を画像上の `groom / bride` 表記付きで並べ、中央に項目名、左右にそれぞれの内容を置く比較表レイアウトです。リアも同様に、左に画像、中央に項目名、右に内容を置く構成です。
`about.html` は、出会いから結婚式当日までを振り返る `Memory Timeline` と、お互いへの質問回答を並べる `Q&A` テンプレートで構成しています。内容は後から差し替えやすい骨組みです。
`gallery.html` は、前撮り写真や今後追加する思い出写真をスマートフォンで見やすく掲載する専用ページとして作り直しており、写真はタップで拡大表示できます。
`agenda.html` は、当日の流れをあとから自由に編集しやすいタイムライン形式のテンプレートです。`seating.html` は、今後追加する席次表画像と、各テーブルの簡単な紹介を掲載できるテンプレート構成です。

## 注意事項

- `photos.json` を更新した後、ブラウザのキャッシュをクリアするか、`cache: 'no-store'` 設定により最新の内容が表示されます
- 画像ファイルは適切にリサイズして、読み込み速度を最適化してください
- サムネイル画像（thumb）を用意することで、ページの読み込みが高速化されます
